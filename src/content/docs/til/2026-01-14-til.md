## 1. Unity MCP 소개

Unity 게임 개발을 원활하게 하기 위한 MCP(Model Context Protocol) 도구를 발견하고 테스트해보았다.

- **프로젝트**: [CoplayDev/unity-mcp](https://github.com/CoplayDev/unity-mcp)
- **목적**: Cursor와 같은 AI 에디터에서 Unity Editor를 직접 제어하여 게임 개발 자동화

Unity Editor와 AI 에이전트를 연결하여 자연어 명령으로 게임 오브젝트 생성, 스크립트 작성, 셰이더 적용 등의 작업을 자동화할 수 있다는 점이 흥미로웠다.

## 2. 설치 및 설정

### Windows 로컬 설정
Windows 환경에서 local setup을 진행했다. Unity Editor에 MCP 서버를 설치하고, Cursor에서 연결하는 구조다.

### MCP 윈도우 토글
Unity Editor 내에서 MCP 윈도우를 토글하여 연결 상태와 로그를 확인할 수 있었다. 실시간으로 Cursor에서 보낸 명령이 Unity에 전달되는 것을 볼 수 있어 디버깅에 유용했다.

### Cursor 설정 (비공식)
공식적으로 Cursor는 지원하지 않지만, `opencode.json` 파일을 수정하여 연결 가능했다:

```json
"unityMCP": {
  "type": "remote",
  "url": "http://localhost:8080/mcp",
  "enabled": true
}
```

비공식 방법이지만 정상적으로 동작했다. 향후 Cursor에서 공식 지원이 추가되면 더 안정적으로 사용할 수 있을 것 같다.

## 3. 테스트 결과

테스트용으로 여러 명령어를 시도해보았다:

### 시도한 명령어들
- **"Create a 3D player controller"**: 3D 플레이어 컨트롤러 생성
- **"Create a tic-tac-toe game in 3D"**: 3D 틱택토 게임 생성
- **"Create a cool shader and apply to a cube"**: 셰이더 생성 및 큐브에 적용

### 실제 동작 경험
AI 에이전트가 Unity Editor를 제어하여 GameObject를 생성하고, C# 스크립트를 작성하고, 컴포넌트를 추가하는 과정을 자동으로 수행했다. 특히 셰이더 생성과 적용은 수동으로 하면 번거로운 작업인데, 한 번의 명령으로 처리되는 것이 인상적이었다.

다만 복잡한 게임 로직의 경우 완벽하게 동작하기보다는 프로토타입 수준으로 생성되었고, 세부적인 조정은 여전히 수동으로 해야 했다.

## 4. 결론

Unity MCP는 게임 개발의 반복적인 작업을 자동화하는 데 큰 잠재력이 있다. 특히 프로토타이핑 단계에서 빠르게 아이디어를 테스트하거나, 보일러플레이트 코드를 생성하는 용도로 유용할 것 같다.

### 장점
- 자연어로 Unity Editor 제어 가능
- 반복 작업 자동화로 생산성 향상
- 프로토타입 빠른 생성

### 한계점
- 복잡한 로직은 여전히 수동 작업 필요
- Cursor 공식 지원 부족 (비공식 방법으로 우회)
- 초기 설정이 다소 번거로움

게임 개발에서 AI 에이전트 활용 가능성을 확인했다는 점에서 의미 있는 테스트였다. 앞으로 Unity 프로젝트를 진행할 때 적극적으로 활용해볼 계획이다.
