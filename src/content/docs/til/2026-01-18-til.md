## 1. 오늘 한 것

- VSCode에 GitHub Copilot 확장 설치 및 실사용 테스트
- Copilot 제안으로 반복 코드(보일러플레이트) 작성해 생산성 확인
- Copilot CLI(`gh copilot`) 간단 테스트
- OpenCode `opencode.json` 설정 확인 및 프리미엄 리퀘스트 영향 실험

## 2. 배운 점 / 느낀 점

- 생산성: 반복적·표준 로직에서 큰 시간 절약을 체감했다.
- 검증 필요: 도메인 특화 로직이나 성능 민감 코드에 대해서는 제안 검증이 필수.
- 툴 선택: VSCode는 실시간 맥락 기반 제안이 강점이고, CLI는 명시적 호출·자동화에 유리하다.
- 비용 주의: OpenCode 등 에이전트 플랫폼에서 전역 모델을 고성능으로 설정하면 사소한 작업에도 프리미엄 모델이 호출되어 비용이 빠르게 소진될 수 있다.
- 권장: 전역은 저비용(또는 비워두기), 프로젝트별 `opencode.json`으로 필요한 에이전트만 고성능 모델 지정, 사용량 모니터링 필수.

## 3. 내일 할 것

- VSCode + Copilot vs Copilot CLI 동일 작업에서 요청 로그·비용 비교 실험
- 프로젝트 루트에 `opencode.json` 두고 브랜치별 모델 정책 테스트
