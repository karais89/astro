## 1. 오늘 한 것

- VSCode에 GitHub Copilot 확장 설치 및 실사용 테스트.
- Copilot 제안으로 보일러플레이트 코드를 작성해 생산성 향상 확인.
- Copilot CLI(`gh copilot`) 간단 테스트.
- OpenCode의 `opencode.json` 설정을 확인하고, 프리미엄 모델 호출이 비용에 미치는 영향 실험.

## 2. 배운 점 / 느낀 점

- 생산성: 반복적·표준 로직에서 큰 시간 절약을 체감함.
- 검증 필요: 도메인 특화 로직이나 성능 민감 코드에서는 제안의 검증이 필수임. 어떤 모델을 선택할지 추가 고민 필요 — GPT-5 mini는 기대 이상으로 성능이 좋음.
  - 큰 작업의 경우 sonet 4.5 또는 opus 4.5 모델 사용 예정.
- 주의 사항: OpenCode에서 Copilot을 연동할 때 서브 에이전트가 동작하면 프리미엄 모델 요청이 발생할 수 있음. 프로젝트별로 `opencode.json`을 설정해 프리미엄 요청 호출을 방지할 것.
- 비교: GitHub Copilot은 이전 사용 경험에 비해 크게 발전했음. 다른 AI 코딩 에이전트와의 차이점과 부족한 부분은 추가 비교 실험이 필요함.

## 3. 내일 할 것

- GitHub Copilot 모델별 성능 테스트와 적합 모델 비교 실험.
