## 1. 오늘 한 것

- SDD(사양 중심 개발) 방법론 학습
- OpenSpec 기반 워크플로우 실습
- OpenSpec 관련 [블로그 글](/blog/2026-01-28-openspec) 작성

## 2. 배운 점 / 느낀 점

- Spec-Kit을 사용했을때는 SDD 방법에 대해 들어보고 설치 이후 너무 복잡하다는 생각이 들어 시도를 해보지 않았으나, OpenSpec의 경우 생각보다 쉬운 접근성을 가지고 있었다. 사양 중심 개발(SDD) 방법론을 처음 적용해보았다. 코드보다 설계 문서를 먼저 만들고, 그걸 기반으로 AI가 구현하는 방식이 신선했다.
- `/opsx:new` → `/opsx:ff` → 문서 수정 → `/opsx:apply` 4단계로 동작한다. 문서 수정만으로 AI 지시를 바꿀 수 있다는 점이 편했고, 작업 진행 상황이 체크리스트로 시각화되는 게 좋았다.
- Copilot skills 연동이 생각보다 직관적이지 않았고, 작은 기능에는 오버헤드가 느껴졌다. 자세한 내용은 [블로그 글](/blog/2026-01-28-openspec) 참고.
