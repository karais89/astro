# H-Index (Lv.2)

### 문제 설명

H-Index는 과학자의 생산성과 영향력을 나타내는 지표입니다. 어느 과학자의 H-Index를 나타내는 값인 h를 구하려고 합니다. 위키백과1에 따르면, H-Index는 다음과 같이 구합니다.

어떤 과학자가 발표한 논문 n편 중, h번 이상 인용된 논문이 h편 이상이고 나머지 논문이 h번 이하 인용되었다면 h의 최댓값이 이 과학자의 H-Index입니다.

어떤 과학자가 발표한 논문의 인용 횟수를 담은 배열 citations가 매개변수로 주어질 때, 이 과학자의 H-Index를 return 하도록 solution 함수를 작성해주세요.

### 제한사항

과학자가 발표한 논문의 수는 1편 이상 1,000편 이하입니다.
논문별 인용 횟수는 0회 이상 10,000회 이하입니다.

## 풀이 코드 (C#)

```csharp
using System;
using System.Linq;

public class Solution {
    public int solution(int[] citations) {
        // 오름 차순 정렬
        Array.Sort(citations);
        Array.Reverse(citations);
        
        int h = 0;
        for (int i = 0; i < citations.Length; i++)
        {
            int candidate = i + 1; 
            if (citations[i] >= candidate)
            {
                h = candidate;
            }
            else
            {
                break;
            }
        }
        
        return h;
    }
}
```

- 인용 횟수를 내림차순 정렬.
- i번째 논문이 i+1편 이상 인용되었는지 확인.
- 조건을 만족하는 가장 큰 i+1 값을 H로 선택.

## 다른 사람의 풀이

```csharp
using System;
using System.Linq;

public class Solution {
    public int solution(int[] citations)
    {
        int answer = 0;
        for (int i = citations.Max(); i > answer; i--)
        {
            if (citations.Count(e => e >= i) >= i)
            {
                answer = i;
            }
        }

        return answer;
    }
}
```

아이디어: 가능한 H값(i)을 큰 값부터 내려가며 직접 검증.

## 학습 메모

- 핵심 패턴: 정렬 + 순위 기반 비교.
- 자주 쓰는 형태: “값 ≥ 인덱스” 또는 “남은 원소 수 ≥ 값”.
- 복습 포인트:
  - Array.Sort 후 Reverse 대신 LINQ로 처리하는 차이.
  - “값-인덱스” 비교 패턴을 활용한 문제 (예: “등수 계산”, “공정한 분배”류 문제).
